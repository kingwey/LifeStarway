# 人生星途（LifeStarway）作品 技术方案

> 赛道：学习工作 + 社会公益
> 产品形态：Web 网站（作品）→ 后续扩展小程序
> 技术栈：Python（后端）+ 前端

---

## 一、Demo 范围定义（MVP 核心功能）

作品 聚焦 Phase 1 MVP 核心路径，实现 **"注册 → 建档 → AI诊断 → 规划生成 → 星图可视化"** 完整闭环，确保评审可体验、可感知产品价值。

### 1.1 包含功能

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 用户系统 | 注册/登录、JWT 认证 | P0 |
| 人生档案 | 个人信息录入（学历、工作履历、技能、性格、薪资）、简历文本导入解析 | P0 |
| AI 诊断引擎 | 职业健康度评估、1年短期+3年中期+5年长期规划生成 | P0 |
| 人生星图 | 里程碑可视化（星空风格）、成长路径连线、达成概率 | P0 |
| 数据沉淀 | 历史数据版本化存储，支持多版本规划对比 | P1 |
| What-If 沙盒 | 假设不同选择，模拟轨迹变化 | P1 |

### 1.2 不包含（后续迭代）

- 微信小程序端
- 付费会员体系
- 第三方简历文件解析（仅支持文本粘贴）
- 对话式 Agent 交互
- 企业版 / 高校合作版

## 三、后端技术方案（Python）

### 3.1 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| Web 框架 | **FastAPI** | 异步高性能，自动生成 OpenAPI 文档，类型安全，生态丰富 |
| ORM | **SQLAlchemy 2.0** (async) | Python 主流 ORM，支持异步，迁移工具成熟 |

| 数据迁移 | **Alembic** | 配合 SQLAlchemy，版本化管理数据库变更 |

| LLM 调用 | **openai** SDK | 兼容 OpenAI / 国内大模型 API，统一接口 |
| 数据校验 | **Pydantic v2** | FastAPI 内置，请求/响应模型自动校验 |
| 配置管理 | **python-dotenv** + Pydantic Settings | 环境变量管理，多环境配置 |

## 四、AI 规划引擎设计

### 4.1 整体流程

```
用户档案数据 ──→ Prompt 构建 ──→ LLM 调用 ──→ 结构化解析 ──→ 存储+返回
                                  ↑
                          Prompt 模板 + 行业知识库
```

### 4.2 LLM 调用策略

- **模型选择**：兼容 OpenAI API 格式，支持国内大模型（如智谱 GLM、通义千问等）
- **结构化输出**：要求 LLM 输出 JSON 格式，通过 Pydantic 模型校验
- **重试机制**：解析失败时重试（最多 3 次），逐步收紧格式约束
- **Token 控制**：使用合理的 max_tokens，规划类任务 2048-4096

### 4.3 核心 Prompt 设计

#### 4.3.1 职业诊断 Prompt

```python
DIAGNOSIS_PROMPT = """
你是一位资深职业规划师，请根据用户的个人档案进行职业健康度诊断。

## 用户档案
{profile_data}

## 诊断维度（每项 0-100 分）
- 成长性：当前岗位的技能成长空间
- 稳定性：职业发展路径的可持续性
- 收入潜力：薪资增长趋势
- 兴趣匹配：工作内容与个人兴趣的吻合度

## 输出格式（严格 JSON）
{{
```

### 5.1 技术选型

| 组件 | 选型 | 理由 |

| 路由 | 页面 | 说明 |

### 5.3 人生星图可视化方案

核心卖点页面，采用 ECharts graph 图 + 自定义渲染：

- **节点（星星）**：每个里程碑对应一颗星，大小=重要性，亮度=达成概率
- **连线（轨迹）**：路径连线用渐变色，主路径粗亮，备选路径细淡
- **背景**：深色星空背景，粒子点缀
- **交互**：hover 显示详情浮窗，点击跳转里程碑详情，可切换主/备路径视角
- **动画**：节点入场渐显、连线流动效果

## 九、关键技术难点与应对

| 难点 | 应对方案 |
|------|----------|
| LLM 输出不稳定/格式错误 | 多次重试 + 格式纠正提示 + Pydantic 强校验 + 降级策略 |
| 星图可视化交互复杂 | 基于 ECharts graph 封装，减少自研渲染逻辑 |
| 简历解析准确率 | 文本分段 + LLM 结构化提取，用户可二次编辑修正 |
| Demo 阶段快速交付 | SQLite 零配置、Docker 单镜像部署、优先核心路径 |
| AI 调用延迟 | 异步任务 + 前端 Loading 状态 + 流式输出(SSE) |
